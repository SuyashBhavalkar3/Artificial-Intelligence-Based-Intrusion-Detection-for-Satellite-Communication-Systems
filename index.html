Technical Contributions

5 Original Research Contributions

Each of these individually could anchor a paper. Together they form a novel system contribution.

CONTRIBUTION 01
Hybrid Cross-Layer Detection Engine
Two parallel detection pipelines fused at output. Layer 1 uses orbital mechanics-grounded signal physics rules (±15dBm, ±50Hz). Layer 2 uses IsolationForest (unsupervised anomaly scoring) + XGBoost (supervised multi-class). Severity is only "critical" when BOTH layers agree — reducing false positives significantly compared to single-layer systems.
Novel Architecture
CONTRIBUTION 02
Post-Quantum Crypto Integration
Kyber-768 (NIST FIPS 203) for key encapsulation and Dilithium3 (NIST FIPS 204) for signatures. The shared secret from Kyber's encapsulation directly becomes the AES-256-GCM key — creating a full hybrid PQC+symmetric pipeline. Per-packet encapsulation provides perfect forward secrecy: each satellite transmission uses a fresh derived key.
NIST 2024 Standard
CONTRIBUTION 03
SHAP-Powered Explainable AI
TreeSHAP computes exact (not approximate) Shapley values for every detection. Operators see precisely which features drove the alert: "85% weight on frequency_deviation, 12% on burst_score." This addresses the critical gap of operator trust in black-box ML systems. LLM (Groq Llama-3.3-70B) then translates SHAP values into natural language incident reports.
Interpretable AI
CONTRIBUTION 04
Blockchain Forensic Notarization
Ethereum smart contract (SatelliteLedger.sol) records SHA-256(src_ip + dst_ip + timestamp) for every detected threat. The contract's require() prevents overwriting — creating an append-only immutable log. A verification endpoint compares local DB state against blockchain — detecting if logs have been tampered post-incident. Critical for legal and military forensics.
Tamper-Proof Audit
CONTRIBUTION 05
5-Class Attack Taxonomy
Most papers do binary (threat/no-threat). SC-IDS classifies into 5 classes: Normal, Jamming (RF interference causing signal degradation), Spoofing (fake signal injection with abnormal gain), Replay (replayed historical packets with low-variance inter-arrival), and DoS (frequency overload + payload spikes). Each has distinct feature signatures grounded in satellite RF physics.
Multi-Class
BONUS CONTRIBUTION
Full-Stack Working Prototype
Not a simulation or MATLAB script — a complete deployable system with FastAPI backend, Next.js 16 frontend, WebSocket real-time streaming, encrypted broadcast channels, fleet management, and an AI chat interface. Research panels increasingly value reproducible, demonstrable systems over theoretical-only papers.
